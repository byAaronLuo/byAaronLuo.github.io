1.4. 问题总结

  1. 目前看效果只能写一次，发送过1次payload之后，是不能向其他目录下写新文件
  2. 发送一次payload之后之后，只能向文件里追加，不会覆盖文件
  3. 如果没有容器化部署（tomcat war包部署），是写不了webshell，虽然还没捋清楚是否是代码执行，如果是代码执行，那么可以在运行态中写入内存马达到作用，如果是命令执行，那么是不行的（看payload的样子感觉是命令执行pipeline）
  4. 如果是以jar包部署，那么考虑到启动jar包的用户权限，一般不会太高，在做了最小权限法的服务器，启动web服务的用户是没有家目录，导致无法写入公钥，也没权限写入计划任务
  5. post包最后也就是第15行后不要有回车，不然生成的文件是shell$\r\n.jsp
  6. 目前看很鸡肋
